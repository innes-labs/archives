GPIO Configuration

SUMMARY

  • The goal of this application note is to explain how to configure available GPIO on Phoenix or Jack connector with a configuration script able to
    • configure GPIO
      • as input or output (Phoenix or Jack35 connector)
        • please refer to installation guide of the used platform to identify the capabilities for each supported GPIO
      • debouncing filter
    • configure Infrared (Jack35 connector)
  • Please see information chapter below to know the supported naming formats

CONFIGURATION FILE: SUPPORTED NAMING

  • These configuration filename format are supported through USB
    • 000000000000.js
    • <MAC>.js (upper case only)
      • ex: 001CE602fc01.js
    • <Mac>.configuration.js (lower case as well as upper case)
      • ex: 00–1C-E6–02-FC–01.js or
      • ex: 00–1c-e6–02-fc–01.js
    • <hostname>.configuration.js (lower case as well as upper case)
    • <UUID>.configuration.js (lower case as well as upper case)

    • configuration.js

      INSTALLATION ERROR MESSAGE

  • An installation error message can be raised by Gekkota in case the configuration file is not compatible with the target player

    INSTALLATION ERROR MESSAGE WHEN INJECTING CONFIGURATION FILE WITH GEKKOTA 3.12.20 (and above)

  • Installation error message can pop with Gekkota 3.12.20 (and above) in case script installation because legacy script JS is containing obsolete command line making it not compatible with this version.
  • 2 possible ways permit to solve the issue
    • Either make compatible your own configuration script js (ex: 000000000000.js) by opening it
      • replacing var logger = logService.getLogger("autoconf"); by var logger = new Logger("autoconf");
      • replacing log("Autoconf start"); by logger.debug("Autoconf start");
    • Or download the script JS 1.10.13 (or above) at the top of this page and configure it again according to your needs
